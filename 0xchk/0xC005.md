### Secure Malware Log Analyzer

Design a C++ program that analyzes malware execution logs while safely handling **runtime errors, malformed inputs, and unexpected data** using **debugging, exceptions, and input validation**.

## üìú Description  

You are building a **malware analysis utility** that reads and processes execution logs generated by malware components.

Since malware data is often **corrupted, obfuscated, or malformed**, the system must:
- Detect and handle **C++ runtime errors**
- Apply **debugging techniques** to trace faulty execution
- Use **exception handling** to isolate failures
- Perform strict **input validation** on log entries
- Prevent crashes when analyzing suspicious data
- Report analysis errors clearly for forensic review

Each log entry contains a **component name** and a **severity score**.

## üìê Validation & Error Rules  

Log entry rules:
- Component name must be **non-empty**
- Severity must be an **integer**
- Severity `< 0` ‚Üí invalid log entry
- Severity `> 10` ‚Üí suspicious overflow condition
- Missing or malformed fields ‚Üí corrupted log format

Error handling:
- Use `try‚Äìcatch` blocks
- Throw custom exceptions for:
  - Corrupted log
  - Invalid severity
  - Overflow condition
- Analyzer must continue processing remaining logs

## üö¶ Output Requirement  

For each log entry:
- Display analysis result:
  - `ANALYZED`
  - `INVALID LOG`
  - `EXCEPTION CAUGHT`
- Display clear diagnostic message
- Do **not** terminate analysis on errors
